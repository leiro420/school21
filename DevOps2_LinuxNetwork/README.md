- ### Устанавливаю **ipcalc** командой `sudo apt install ipcalc`
    1. #### Сети и маски. Определи и запиши в отчёт:
    - Адрес сети *192.167.38.54/13* = 192.160.0.0 \
    ![ipcalc1](images/png01-1.png) 
    - Перевод маски *255.255.255.0* в префиксную = /24 \
    и двоичную запись = 11111111.11111111.11111111.00000000, \
    ![ipcalc2](images/png01-2.png)
    - */15* в обычную = 255.254.0.0. \
    и двоичную = 11111111.11111110.00000000.00000000, \
    ![ipcalc3](images/png01-3.png)
    - *11111111.11111111.11111111.11110000* в обычную = 255.255.255.240 \
    `Из-за того, что двоичную форму адреса передать на вход ipcalc нельзя мы 11111111.11111111.11111111.11110000 переводим в обычную 11111111 = 255.` \
    `Вычисляем 11110000: 2^7 + 2^6 + 2^5 + 2^4 = 128 + 64 + 32 + 16 = 240` \
    и префиксную = /28 (просто складываем все единицы)
    - Минимальный и максимальный хост в сети *12.167.38.4* при масках: 
        + /8: HostMin: 12.0.0.1, HostMax: 12.255.255.254 \
        ![ipcalc5](images/png01-5.png)
        + *11111111.11111111.00000000.00000000* (складываем единицы, получаем префикс /16), \
        /16: HostMin: 12.167.0.1, HostMax: 12.167.255.254 \
        ![ipcalc6](images/png01-7.png)
        + *255.255.254.0* (переводим в двоичную и складываем 8 + 8 + 7 = 23), \
        /23: HostMin: 12.167.38.1, HostMax: 12.167.39.254 \
        ![ipcalc7](images/png01-8.png)
        + /4: HostMin: 0.0.0.1, HostMax: 15.255.255.254 \
        ![ipcalc8](images/png01-6.png)

    2. #### localhost

    Можно только с двух адресов: **127.0.0.2**, **127.1.0.1** (имеют параметр Loopback)

    5. #### Построение списка маршрутизаторов
    - Запускаю на r1 команду `tcpdump -tnv -i enp0s8` \
    ![network25](images/png05-20.png)
    - Пакеты TTL успешно достигли хоста, ответ ICMP подтверждают получение хостом пакета, и ARP запросы тоже достигли хоста \
    - На ws11 запускаю команду до ws22 `traceroute 10.20.0.20`
    ![network24](images/png05-19.png)
    - Путь успешно построен: 
        - Hop 1: Шлюз 10.10.0.1 (r1) 
        - Hop 2: Маршрутизатор 10.100.0.12 (r2) 
        - Hop 3: Конечный хост 10.20.0.20 (ws21)
    - Traceroute строит путь, последовательно "прощупывая" каждый хоп с помощью пакетов с увеличивающимся TTL. Ответы ICMP Time Exceeded от маршрутизаторов раскрывают цепочку промежуточных узлов, а ICMP Port Unreachable от конечного хоста сигнализирует о завершении трассировки. ARP обеспечивает преобразование IP-адресов в MAC-адреса на канальном уровне, без чего невозможна доставка пакетов между соседними узлами
